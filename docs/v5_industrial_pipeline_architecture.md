# 智能分镜系统研发记录 V5.0：工业级多节点制片管线 (Multi-Node Production Pipeline)

## 1. 架构突破：摒弃“盲盒式”一站通关
在 V4 (HITL 单断点管线) 中，我们在剧本解构与生图引擎之间插入了人类审核锁。但导演/制片人指出，这**依然不够工业化**。
真正的影视工业管线是高度结构化的流水生产。如果人类只能在“分镜表格”那一刻做唯一一次审查，系统往往会在人物设定或整体色彩基调这种早期大方向上出错。

因此，V5 将生图的“原子操作”重新定义为一条长周期的、包含 **5 次上下文串联、5 次人类拦截查验 (Multi-stage HITL)** 的终极异步状态机系统 (FSM)。

## 2. 五步工业流转线 (The 5-Step Industrial Protocol)

### 节点 1: 剧本大纲与情绪充实 (Script & Emotion Expansion)
* **人类起手式**：提供基础故事线（如：主角走在废土街头）。
* **Orchestrator 动作**：挂载【剧情认知包】，补全人物心理状态、梳理环境光流、设计情绪点（钩子）。
* **⏸️ HITL 锁 1**：导演不看任何分底或图片，只审查这段充实后的文字文本。修改并定稿后，此文本作为后续阶段的 **Ground Truth Context Level 1** 写入 Memory Server。

### 节点 2: 人物小传与美术设定图 (Character & Concept Design)
* **Orchestrator 动作**：将定稿剧本脱水，梳理出主要出场人物和核心关键场景。并发调用 Prompt Maker 生成人物三视图或白模场景需求。
* **执行工具**：调用生图引擎跑出概念图。
* **⏸️ HITL 锁 2**：导演在这个环节面对画面。挑选一张满意的人物正脸图与场景图，勾选通过。
* **上下文回流合并**：选中的图片及其描述，被打包作为 **Reference ID (如 FaceID / Style Adapter)** 写入 Memory Server (Ground Truth Level 2)。

### 节点 3: 制作文字分镜脚本 (Text Storyboard Breakdown)
* **Orchestrator 动作**：携带前面所有的剧本和美术资源，开始干脏活。将长文剧本彻底碾碎，分解成 Shot 01 ~ Shot N 的独立表格卡片。自动为其计算摄影短焦长焦、机位景别。
* **⏸️ HITL 锁 3**：也就是我们在 V4 原型里做的那张网格工作台，导演做精装修。修改完毕推入总数据库 (Ground Truth Level 3)。

### 节点 4: 粗略预演与灯光模拟 (Blockout & Lighting Pre-viz)
* **Orchestrator 动作**：这是全管线最省算力的核查区。引擎（如通过 ComfyUI）快速生成不包含高清人脸肌理的灰模底图或线稿图。只用以确定机位构图和主体光照方向（如大逆光）。
* **⏸️ HITL 锁 4**：导演若发现空间崩塌（比如上个镜头门在左边，这个镜头门变右边了），直接在此阶段打回或重置景别参数，避免了到渲染环节才发现的昂贵废片。

### 节点 5: 多参考最终合成渲染 (Multi-reference Final Comp)
* **Orchestrator 动作**：调集全部武装。拿着阶段 2 的 FaceID（人物长相）、阶段 3 的 Prompt 镜头（焦段细节）以及阶段 4 的预演底图（作为 ControlNet Depth 或 Canny 约束），向生图集群发起源源不断的请求。由于全都是强约束，成片率达到极致工业标准。
* **⏸️ HITL 锁 5**：成片展现于瀑布流。容许导演使用外接 PS 工具微调，终版归档签发。

## 3. 架构结论
V5 完全摆脱了 AIGC 产品流行的“聊两句就丢一堆破图”的快餐模式。把 Orchestrator 从一个魔法师变为了一个循规蹈矩的“执行制片人 (Line Producer)”。每一环节人类导演都在修桥铺路，AI 只是高效执行那些确定的基建作业。这是当下 AIGC 进入商用影棚的最强定海神针。
