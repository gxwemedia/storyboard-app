## 项目概述

**项目名称**: 智能分镜制片台 (Storyboard App)
**版本**: V7.0 - 认知技能系统与多智能体编排基座

## 技术栈详细分析

### 前端框架层
- **React 19.2.4**: 支持并发特性，优化渲染性能
- **TypeScript 5.9.3**: 全量类型覆盖，提高代码质量
- **Vite 7.3.1**: 极速开发体验，支持热重载

### UI组件层
- **Radix UI**: 提供无样式、可访问性基础组件
- **shadcn/ui**: 基于Radix的高质量预配置组件
- **Tailwind CSS 4.2.1**: 原子化CSS，快速样式开发
- **Framer Motion 12.35.0**: 声明式动画，流畅交互体验
- **Sonner 2.0.7**: 现代化Toast通知组件

### 状态管理
- **Zustand 5.0.11**: 轻量级状态管理，替代Redux
- 优点：无需Provider，TypeScript友好，Middleware支持

### 数据验证
- **Zod 4.3.6**: 运行时类型校验，确保数据一致性
- 用于：ShotSpec JSON验证，API响应验证

### 开发工具
- **Vitest**: 快速测试框架，Vite原生集成
- **Lucide React**: 现代化图标系统
- **class-variance-authority**: 条件类名工具

## V7架构创新点

### 1. Stage重构
- 从6阶段压缩为5阶段 (0-4编号)
- Stage 0合并了原来的Stage 0和Stage 1
- 每个阶段提供手动AI触发按钮，导演保留控制权

### 2. 认知技能系统
- 对齐Claude/Codex CLI的Skills范式
- 5分类技能包：bible/script/concept/shotspec/rendering
- 技能格式：YAML frontmatter + Markdown指令
- 技能注入到AI system prompt，提供领域知识

### 3. 多智能体编排优化
- 每个阶段有专属智能体
- 智能体支持技能指令注入
- 上下文感知，前序阶段结果自动继承

### 4. 工业级特性
- **资产血缘追踪**: 构建DAG图，跟踪依赖关系
- **变更传导**: 自动分析修改影响范围
- **Stale标记**: 识别陈旧资产，支持局部恢复
- **MCP协议**: 标准化的AI服务接口

## 系统交互流程

### 导演工作流
```
1. 创建新项目 → 进入Stage 0
2. 填写项目圣经 → 锁定世界观
3. 输入剧本大纲 → AI扩写(可选)
4. 签发Stage 0 → 进入Stage 1
5. 选择概念技能 → AI生成概念图
6. 逐阶段推进 → 最终Stage 4
7. 批注确认 → 导出交付包
```

### AI辅助流程
```
1. 用户选择技能包
2. 系统加载技能指令
3. 注入到Agent system prompt
4. 调用AI API (通过Orchestrator)
5. 处理AI响应
6. 结构化输出到工作台
7. 导演审查并修改
```

### 数据流转
```
JSON数据 → Zod验证 → Store更新 → UI渲染
↓
用户修改 → Store更新 → 变更传导 → 血缘DAG更新
↓
AI调用 → API请求 → AI响应 → 结果处理 → Store更新
```
