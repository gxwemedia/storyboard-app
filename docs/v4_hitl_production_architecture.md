# 智能分镜系统研发记录 V4.0：商业化大势——HITL 拦截管线终局架构

## 1. 架构定位 (Architecture Positioning)
这不仅是一份工程手册，更是一场基于实际生成式 AI（AIGC）算力经济学的降维思考。
当系统推进到 V3.0 (认知解耦与 MCP Re-Act 全自动回环) 时，我们在纯技术极客眼里算是做到了极致：大模型能自己挂载技能、自己分解结构、自己请求服务器、甚至画错了自己校验打回重做。
但这在工业流中是绝对的**灾难设计 (Full Autonomy Disaster)**，原因很简单：大算力很贵（比如调用集群跑 ComfyUI 及 ControlNet 等多管线任务），若 AI 一开始（剧本到 Prompt）发生方向型误判（比如将恐怖悬疑的调子解析成小清新），那么它后面全自动跑出来的 30 个分场大片，全都是高质量的垃圾。

为此，**V4 引入了必将伴随 Agent 到永久的特性设计——人机协作环路 Human-in-the-Loop (HITL)**。

## 2. HITL 物理介入层 (The HITL Intervention Layer)
在系统最关键的分水岭（认知逻辑向物理渲染引擎妥协的节点），物理层面强行打下了一个“桩”：它赋予了人类拥有着最高层级的生杀大权——最终剪辑特权(Final Cut)。

### 2.1 分水岭挂起 (Suspend at the Watershed)
Orchestrator（中央主脑）在使用挂载的【顶层：编剧技能包】完成了整套文字版故事框架搭建与摄影机位规划后。
*   **强制状态切换**：后台代码触发中断逻辑（例如通过 Promise 或 Generator yield机制），将自己标记为 `SUSPENDED (挂起待唤醒)`。
*   **指令静音**：严禁在这个状态下，调用底下哪怕一个 MCP Server API。

2. **推拉博弈式人类交互 (Push and Pull Interactions)**
挂起后，系统将 AI 编写好的那份“绝对理性但缺乏灵魂火花”的分镜结构阵列（JSON 态文本），推向人类导演的 Web UI 看板。这里面，支持三种操作：

1. **记忆回流式修改权 (Contextual Edit & Approve)**
    * 场景：AI 设定这里是（Extreme Wide Shot, 全景仰角，压抑基调），导演看后认为方向不错，但希望细节更具体，于是直接修改为（Close-up, 面部特写，狰狞眼神，且补充了一句“角色拔出了他的传家宝剑”）。
    * 动作：导演在可视化的 JSON 操作台上进行微调或增加剧情设定。
    * 结果：当导演签发时，系统不仅按照修改后的指令去生成当前帧，**最关键的是：这份被人类“开过光”的修改后剧本，会被作为最新的“真相上下文 (Ground Truth Context)”回流动步至 `Script & Memory Server`！** 在此后的第 2 场、第 3 场戏的创作中，大模型都会“记住”并沿用导演刚才添加的设定（例如那把传家宝剑、角色心态的转变等）。这就是真正的“同频演进”。
2. **打回重做权 (Soft Re-roll Penalty)**
    * 场景：AI 完全写崩了高潮前戏。
    * 动作：导演给出整体批注“悬疑不够，请从 Shot 03 重画”。
    * 结果：系统剥离生图管线，把历史语境连同批评的话一起塞回原点。 Orchestrator 灰溜溜地再次调用【技能包】来重铸剧本。
3. **彻底授权 (Release to Render)**
    * 场景：完美的分镜文字方案被彻底敲定。
    * 动作：人类签发“释放执行锁”（相当于在单据上按手印）。系统恢复并同时并发下发海量生图 Request 给底下的工具库。

## 3. 终局系统的全链路拓扑图 (End-Game Topology)
此版本的 MCP + HITL 双轴结构彻底奠定了产品从“玩具模型”进军“影棚专业副驾 (Copilot)”的技术壁垒：

1. `用户给出一句话` -> **[认知技能挂载]** Orchestrator 利用顶层 RAG 常识包自动脑补撰写详情。
2. `文字拆解 JSON 流出` -> **[触发 HITL 人机断点]** 系统全局挂起，前端弹窗等待干预。
3. `人类持续共创与上下文回放 (Context Update)` -> 既然 AI 的方向不错，导演可以直接在它的基础解构上修修补补，随后将其保存。这不仅决定了当下的生图命令，也顺带刷新了全局的剧本记忆长文。
4. `执行锁打通 (Proceeding...)` -> Orchestrator 收到回执，开始疯狂并发呼叫底部四个 MCP Servers。
5. `记忆池 Server 存取` -> 获取持久层的连贯人脸特征与导演刚刚更新的动作细节。
6. `Prompt / 渲染 Server 打底图` -> 根据人类的定版指令重制或搭建基于景深控制的立体空间墙。
7. `CV 环境 Server 检查` -> 取代人眼做 24 小时不间断的废片筛查。
8. `图元交付` ->  **全剧终。一张具备极强一致性的，打着顶级悬疑/奇幻光影的分镜图震撼呈现。**
